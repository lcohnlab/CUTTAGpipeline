# CUT&Tag Snakemake Pipeline

**Automated pipeline for processing CUT&Tag data following the Henikoff lab tutorial (https://yezhengstat.github.io/CUTTag_tutorial/index.html)**

The pipeline uses Snakemake workflow to automate CUT&Tag preprocessing. From raw FASTQ files as input (paired-end CUT&Tag data), the pipeline performs alignment, filtering, fragment processing, spike-in normalization, SEACR peak calling, and generates a peak-associated count matrix for downstream analysis.

## Getting Started

### Input Files

- ```data/samples.tsv```

This file should contain your sample metadata. It must contain the following columns: [sample, filename, path]. *sample* is the sample name (sample1), *filename* is the fastq filename (sample1_R1.fastq.gz), and *path* is the path to the file (~/path/to/project/CUTTAGpipeline/fastq/)

- ```config.yaml```

This file stores global settings and paths used in the pipeline. These need to be updated prior to running to match correct samples and indexes.

Default entries include:

```
projPath: "~/path/to/project/CUTTAGpipeline"
fastq_dir: "~/path/to/project/CUTTAGpipeline/fastq/"
alignment_dir: "alignment"
chromSize: "~/path/to/hg38.chrom.sizes"
hg38_index: "~/path/to/bowtie2Index/hg38"
SPIKEIN_INDEX: "/path/to/bowtie2Index/[spikein-prefix]"
picard_cmd: "java -jar $EBROOTPICARD/picard.jar"
seacr_script: "~/path/to/project/CUTTAGpipeline/SEACR/SEACR_1.3.sh"
histControl: "IgG_rep1"
```

### Set-Up

1. Copy files

Clone the base files from this repository and additionally clone SEACR. Either move raw-reads to fastq/ folder or update the paths to read from correct location. The directory should generally have a similar set up to what is described below.

2. Update config.yaml and samples.tsv

Ensure that the files above are udpated to match correct paths and sample names.

3. Verify software availability

Make sure you have the require software available (see below). 

4. Dry-run snakemake

```snakemake -n -p```

Perform a dry-run of snakemake to check the workflow prior to execution

5. Run snakemake

```snakemake -p --cores 8```

Adjust the core count to match available resources (at least 8 is recommended)

## Directory Structure

The standard project layout should match this structure

```
CUTTAGpipeline/
├── Snakefile
├── config.yaml
├── data/
│   ├── results/
│   └── samples.tsv
├── fastq/
│   ├── sample1_R1.fastq.gz
│   └── ...
├── alignment/
│   └── ...
├── fastqFileQC/
├── peakCalling/
│   └── SEACR/
└── logs/
```
Folder descriptions:

1. data/: sample metadata and input tables
2. fastq/: raw FASTQ files
3. alignment/: alignment-derived outputs (generated by pipeline steps)
4. fastqFileQC/: FastQC reports
5. peakCalling/: SEACR and SEACR outputs
6. logs/: log files (generated by snakemake)

## Requirements

Ensure the following tools/libraries are available in your environment

- Snakemake
- FastQC
- Bowtie2
- Samtools
- Bedtools
- Picard
- SEACR
- R (dplyr, GenomicRanges, chromVAR, rtracklayer, Rsamtools, DESeq2)

### SEACR

[SEACR](https://github.com/FredHutch/SEACR/) must be available as a local clone. In config.yaml, define the path to the SEACR shell script, for example:
```
seacr_script: "/path/to/SEACR/SEACR_1.3.sh"
```

## Expected outputs

After the workflow completes, you should have:

- FastQC reports in fastqFileQC/
- Bowtie2 SAM/BAM files in alignment/sam/ and alignment/bam/
- Fragment BED files in alignment/bed/
- Spike-in normalized bedGraphs in alignment/bedgraph/
- SEACR peak calls in peakCalling/SEACR/
